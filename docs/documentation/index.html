Inner workings

Listed below is some useful information that can help you understand how Onyx functions.

The code

The code used to build Onyx is designed to be as generic as possible. It is possible to turn Onyx into a completely different virtual being in a few short minutes.

Data flow

Onyx is especially rigged to make data transfer between services as efficient as possible. This is crucial as Onyx uses several web services like API.AI, Firebase and Heroku in tandem with one another.

How API.AI connects to Facebook Messenger / Twitter

When you text Onyx, Facebook / Twitter records your message and sends it to Heroku, where Onyx's Node.js app is deployed. This app then transfers your message to API.AI as a request. The resulting information then retraces the exact same path.

How API.AI connects to Onyx for Android

This is a fairly simple process. Onyx for Android uses the SDK provided by API.AI to directly connect to and transfer data between the service and the app.

Known issues

• Onyx for Facebook Messenger, Twitter, Slack and Google Assistant does not store any information on Firebase, correctly parse API.AI's response or react to emoji.

• Onyx for Android fails to successfully serve a returning customer due to a recurring bug that prevents the app from collecting the recipient's phone number.

• Due to lack of data to fuel Machine Learning, Onyx tends to feel quite machine-like, a serious setback for a chatbot that aims to feel as human as possible.
